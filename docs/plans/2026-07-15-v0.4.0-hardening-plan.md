# Lvau 0.4.0 implementation plan

Date: 2026-07-15

## 1. Lock in security regressions

- Add a failing test that rewrites `plaintext_len`, retains a valid ciphertext prefix, and demonstrates the version 1 truncation behavior.
- Add failing tests for empty-payload metadata mutation and trailing ciphertext.
- Add malformed/short/oversized envelope tests for the CLI-facing parser.
- Run only the focused tests and retain their failing output before implementation.

## 2. Add format v2 and the bounded compatibility parser

- Set the current writer version to 2 while accepting versions 1 and 2.
- Add an exact legacy 0.2.x envelope decoder and a release-generated compatibility fixture.
- Centralize envelope framing and remove unchecked CLI slicing.
- Define and document the domain-separated v2 payload commitment.
- Check serialized envelope lengths on both write and read paths.

## 3. Authenticate payload length and termination

- Pass the format version into streaming payload encryption/decryption.
- Emit and verify an authenticated empty frame for zero-length v2 payloads.
- Reject suffix bytes after the expected ciphertext.
- Keep the version 1 read path byte-compatible.
- Re-run focused password, key-pair, memory, truncation, and empty-file tests.

## 4. Bound KDF and recipient work

- Validate Argon2id parameters against the declared profile before derivation.
- Reject missing, mixed, or empty recipient configurations.
- Make key-pair verify and in-memory decrypt try every recipient consistently.
- Add unit tests for malicious KDF values and second-recipient success.

## 5. Harden temporary files and secret files

- Replace hand-built plaintext temporary paths with RAII same-directory files.
- Preserve atomic replacement on Unix and sync the containing directory.
- Use atomic owner-only writes for signing keys and recovery shares.
- Add cleanup and Unix mode regression tests.

## 6. Correct signature identity and approval checks

- Derive the displayed fingerprint from the supplied verification key.
- Bind v2 signature metadata to the signature statement while retaining v1 verification.
- Recompute the approval subject before accepting approvals.
- Add tampered-fingerprint, metadata, and approval tests.

## 7. Validate bundles deeply

- Add checked manifest range and collision validation.
- Verify every entry hash in `verify_bundle`.
- Validate the entire manifest before writing extracted files.
- Preserve the user's existing `NamedTempFile`/`tempdir` improvements.
- Add overflow, overlap, hash mismatch, and no-partial-output tests.

## 8. Align CLI, GUI, docs, and versioning

- Bump workspace crates and visible product references to 0.4.0.
- Make inspect/preflight explicitly label legacy v1 and unverified attestations.
- Update README, format, security, threat model, roadmap, changelog, approvals, policy, release notes, and examples to match tested behavior.
- Add root and focused `AGENTS.md` guidance.

## 9. Harden CI and release preparation

- Add least-privilege permissions, locked Cargo commands, cross-platform tests, version/tag validation, checksums, SBOM, and artifact attestations.
- Pin actions to reviewed full commit SHAs with version comments.
- Remove permanent advisory ignores or document narrow, expiring exceptions.
- Validate workflow syntax locally where tooling is available.

## 10. Align the related site and OCI API

- Update `lattes.jp` download links, displayed version, checksums/verification guidance, metadata, and accessibility without changing its established visual system.
- Diagnose and correct the `lvau-api` auth/rate-limit ordering tests, format/clippy failures, deployment docs, health checks, and secret handling.
- Keep OCI commands dry-run/documentation-only; do not deploy or mutate cloud resources.

## 11. Verify and hand off

- Run `cargo fmt --all --check`.
- Run `cargo clippy --workspace --all-targets --all-features -- -D warnings`.
- Run `cargo test --workspace --all-features`.
- Run `cargo build --workspace --release`.
- Run dependency/security checks available in WSL.
- Run site/API tests, lint, and production builds.
- Review final diffs and statuses in all touched repositories.
- Report commands, results, residual risks, migration guidance, and the exact manual tag/release/deploy steps without executing them.
