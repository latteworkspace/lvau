Lvau は、最先端の E2EE (エンドツーエンド暗号化) を堅牢かつ簡単に利用できるように設計された次世代の暗号化ツールです。独自の難読化には依存せず、モダンで安全なデフォルト設定に厳密に準拠しています。

## ✨ 主な機能

### 🛡️ 鉄壁の「退屈な」セキュリティ
- **XChaCha20-Poly1305 AEAD**: デフォルトの暗号化アルゴリズム。ファイルごとに新しく192ビットのランダムなノンスを生成し、ノンスの再利用を完全に防止します。
- **Argon2id KDF**: マスターキーを導出するための強化されたパスワードハッシュ。暗号化ごとに一意の16バイトのランダムソルトを使用します。
- **HKDF による鍵分離**: パスワードから導出されたマスターキーは、ファイル暗号化キーへと安全に拡張されます。
- **秘密情報のゼロ化 (Zeroized Secrets)**: 生のパスワード、導出されたキー、および平文は、使用後直ちにメモリから消去されます。
- **バージョン管理された `.lvau` エンベロープ**: AEAD の追加認証データ (AAD) を介して、すべての暗号化メタデータをバインドする強力に型付けされたエンベロープスキーマ。

### 🧩 未来を見据えたアーキテクチャ
Lvau は、将来の技術的進歩をサポートするように構造設計されています。`lvau-protocol` には以下のためのプレースホルダーがあります：
- **非対称 E2EE**: 受信者キーのラッピング用の `X25519` と、マニフェスト署名用の `Ed25519`。
- **耐量子暗号 (PQC)**: `ML-KEM` および `ML-DSA` ハイブリッド。
- **運用鍵プロバイダー**: KMS/HSM 抽象化インターフェース。*(注: ローカルファイルの E2EE は常にローカルで暗号化されます)*。

*(注: Lvau の安全性に関する哲学に基づき、独自の暗号や「極端な難読化レイヤー」はセキュリティ境界から意図的に除外されています)*。

## 🚀 インストール方法

### 前提条件
プロジェクトをビルドするには、[Rust と Cargo](https://rustup.rs/) がインストールされている必要があります。

### ビルド手順
```sh
# リポジトリをクローン
git clone https://github.com/yourusername/lvau.git
cd lvau

# ワークスペース全体をリリースモードでビルド
cargo build --release
```

## 📖 使い方

Lvau は **CLI** または **GUI** 経由で使用できます。

### CLI の使い方 (`lvau-cli.exe`)

CLI は、シェルの履歴への漏洩を防ぐため、パスワード入力時に非表示のプロンプトを使用します。

**1. ファイルの暗号化**
```sh
lvau-cli encrypt --in secret.txt --out encrypted.lvau --profile balanced
```

*利用可能なセキュリティプロファイル: `fast`, `balanced`, `archive`, `paranoid` (Argon2id のコストが直線的に増加します)*。

**2. ファイルの復号**
```sh
lvau-cli decrypt --in encrypted.lvau --out secret.txt
```

**3. エンベロープメタデータの検査**
コンテンツを復号せずに、AAD メタデータを読み取ります。
```sh
lvau-cli inspect --in encrypted.lvau
```

### GUI の使い方 (`lvau-gui.exe`)
クロスプラットフォーム対応のネイティブなグラフィカルウィザードが利用可能です。対象のファイルを選択し、パスワードを定義して実行するだけです。テレメトリーは安全なメタデータのみを厳密に表示し、平文チャンクの露出を防ぎます。

## 🏗️ ワークスペース構成

Lvau は、攻撃対象領域 (アタックサーフェス) を最小限に抑えるため、独立したクレートにモジュール化されています。
- `lvau-protocol`: バイナリフォーマットの定義とシリアライゼーション (`postcard`)。厳密な `Envelope` 仕様を含みます。
- `lvau-core`: `secrecy` クレートによる制約のもとで、Argon2id、HKDF、および XChaCha20-Poly1305 ロジックを処理する暗号化エンジン。
- `lvau-cli`: `clap` によるコマンドラインインターフェース。
- `lvau-gui`: `egui` で構築されたハードウェアアクセラレーション対応の GUI。
- `lvau-stub`: 将来の自己解凍形式 (SFX) 統合機能のための最小限のプレースホルダー実行ファイル。
